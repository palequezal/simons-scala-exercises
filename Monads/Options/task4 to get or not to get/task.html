Task 4: To get, or not to get?

Let me preface this by saying it's always "not to get()"! Always!

That requires some explanation...


The get() method

The Option Monad has a devious, sinister, evil method called get() which you should never ever use.

It's purpose is to trick you into thinking it's a good idea, purely because it exists. It is not.

The get() method ignores the fact that an Option is optional and returns you the contents of the Option.

But what, I hear the eagle-eyed amongst you saying, happens if the Option is a None?

It breaks... That's what... Almost as if we weren't using Options at all.

"OK, OK, I get it", I hear you saying, "But what should I use instead?"


The getOrElse() method

getOrElse() also gives you the contents of the Option, but with one critical difference... It allows you to define a default if the Option doesn't contain anything.

This is much safer because with get() you're making some hefty assumptions that the value you've said will be optional will always be there. Seems sort of counter-intuitive, don't you think?
