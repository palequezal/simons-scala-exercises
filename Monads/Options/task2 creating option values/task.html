Task 2: Creating Option values

We now have the ability to work with an existing Option, but how would we go about making our own?

The Option[T] monad has two sub types in Scala, Some and None

Some

Some has the semantic meaning that the value is present. You create a Some value using it's apply() method, which can be done like either example below, both are equivalent

  1. Some(???)
  2. Some.apply(???)

... Where "???" is whatever you want to put inside the option. The type of the contents will be inferred from whatever you put inside, so Some(1) has the type Option[Int] and Some("Hello") has the type Option[String]

By convention we tend to use Some(), rather than Some.apply() because it's more readable, but it's useful to know what's happening under the hood, should you need it at some point. Some point, get it?


None

None represents the absence of a value. It doesn't take a type, but None works with any Option as under the hood it's technically Option[Nothing] and Nothing is a subtype of every type.
